Install GCC-XML with one of these approaches:
  • On Windows, you can use the pre-compiled installation binary. See the Download page to get it.
  • On UNIX, you need to build GCC-XML yourself from the pre-patched source to GCC 3.0. See the Download page to get it. This is also an option for cygwin users, although the binary version will work.
  • On UNIX, you can install GCC 3.0 with the extension available as part of the compiler. See the page on Patching GCC for instructions.
The rest of these instructions assume you have chosen to build GCC-XML yourself. It is also assumed that you have the GCC source (with patch applied) in the directory ~/gccxml. You will build the program in a parallel directory called ~/gccxml-build.
  1. Configure and build GCC (but don't install it). Go here for instructions. Don't bother with a bootstrap, just run make after the configuration step. Example:
    $ cd ~
    $ mkdir gccxml-build
    $ cd gccxml-build
    $ ../gccxml/configure --enable-languages=c++ --disable-nls
    $ make
    This example builds all of GCC's C++ compiler and its standard library, but all you really need is one exectuable. If you want to try to build the minimal portion of GCC needed, try these instructions.
  2. When the build finishes, find the file ~/gccxml-build/gcc/cc1plus. This program is GCC's C++ front-end which is run by the standard gcc executable for C++ programs. It does not automatically add any preprocessor macros (like __GNUC__), so we can use it to exactly control how our source is parsed. This is the only part of the compiler you need, and is the gcc-xml executable. Copy this program to wherever you want to install it, and rename it to gccxml.
