The installation process for GCC-XML is different on Windows and UNIX. The Windows installation is simpler than in UNIX, but is also much less flexible.

Windows

There is a pre-compiled copy with an installation program available. Refer to the Download page to get it. The installer will automatically take care of the compiler-specific support library installation. Currently support is only available for C/C++ headers provided by Visual C++ 6.0 (tested with service pack 5). See the page on Running GCC-XML for details about this support library.

UNIX & Cygwin

Installation on a UNIX platform is a two-part process:
  1. Installation of the GCC-XML executable.
  2. Installation the compiler-specific support libraries.

Building and Installing the GCC-XML Executable

There are several ways to build the executable.
  • Download the pre-patched source to GCC 3.0 and build it. See the Download page to get it. This is the recommended approach for UNIX users, and is detailed below.
  • Download the entire GCC 3.0 compiler source from GCC's web site, apply the GCC-XML patch, and install the compiler with the extension built in. See the page on Patching GCC for instructions.
  • Cygwin users can use the Windows installer program to get the binary pre-built. When prompted for component selection, just disable the Visual C++ 6.0 support if you don't want it.
Follow the instructions below to build the part of GCC you need to get the GCC-XML executable. It is assumed that you have extracted the pre-patched GCC source in your home directory, so the source appears in ~/gccxml. You will build the program in a parallel directory called ~/gccxml-build.
  1. Configure and build GCC (but don't install it). Don't bother with a bootstrap, just run make after the configuration step. Example:
    $ cd ~
    $ mkdir gccxml-build
    $ cd gccxml-build
    $ ../gccxml/configure --enable-languages=c++ --disable-nls
    $ make
    This example builds all of GCC's C++ compiler and its standard library, but all you really need is one exectuable. If you want to try to build the minimal portion of GCC needed, try these instructions. If you want more detailed instructions about building GCC, go here.
  2. When the build finishes, find the file ~/gccxml-build/gcc/cc1plus. This program is GCC's C++ front-end which is run by the gcc driver program for C++ sources. It does not automatically add any preprocessor macros (like __GNUC__), so we can use it to exactly control how our source is parsed. This is the only part of the compiler you need, and is the gcc-xml executable. Copy this program to PREFIX/bin, and rename it to gccxml. Here, "PREFIX" refers to your GCC-XML support library installation prefix, such as /usr/local (see below).

Installing the compiler-specific support libraries

The support libraries contain scripts and headers that help you run GCC-XML to parse your code just as your compiler would. See the page on Running GCC-XML for details on these libraries.

To install these, you will need to have a CVS checkout of GCC_XML. See the Download page for details. In the root directory of the GCC_XML source distribution, there is a Makefile whose only purpose is to install the support libraries. From that directory, run "make" for more instructions. The compiler support libraries you select will be installed in PREFIX/share/GCC_XML. There will be a subdirectory for each supported compiler. In many cases, you will need to add the appropriate subdirectory to the beginning of the include path when GCC-XML is run.
