Running GCC-XML has the goal of providing an XML description of your code as it is seen by the compiler when it is built. This means that GCC-XML should make use of the same standard library and other headers as your compiler. The hard part about running GCC-XML involves determining the set of -I and -D options it needs to use the headers from the compiler of your choice.

Run gccxml with the -fxml=some-file-name flag (added by extension) to enable the XML output after parsing. Try running gcc-xml on this example source:

$ cc1plus -quiet -nostdinc -o /dev/null -fsyntax-only example1.cxx -fxml=example1.xml
If there are no errors, the XML description of the program will be written to a file with the provided name. For this example, it looks like this. Note that the options used above are not sufficient for much more than the example given. If any files are #included, more options will be needed.
